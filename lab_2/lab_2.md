# Лабораторная работа № 2. RAID, LVM, NFS, cron, файрвол и сетевые сервисы

**Среда:** VirtualBox или VMware. **Образ:** Ubuntu Server. **Одна ВМ** на всю работу; вторую ВМ поднимать не нужно. Проверки части 2 — с той же ВМ, с хоста.

**Требование:** при настройке RAID, LVM, NFS, cron, файрвола и сетевых сервисов необходимы скриншоты или выводы ключевых команд.

---

## Задания

**Часть 1**

1. Развернуть виртуальную машину из образа Ubuntu Server (или использовать ВМ из лабораторной работы № 1). Подключить к ВМ не менее трёх дополнительных виртуальных дисков для RAID 5.

2. Настроить SSH-сервер на виртуальной машине: доступ пользователю с правами администратора по ключу, отключить парольную аутентификацию. Дальнейшую работу выполнять в рамках SSH-сессии.

3. Настроить сетевой интерфейс виртуальной машины: добавить при необходимости второй адаптер в гипервизоре, настроить статический IP через netplan, зафиксировать выводы `ip a`, `ip r` и при необходимости `ss -tuln`.

4. Изучить конфигурацию дискового пространства виртуальной машины (диски, разделы, свободные устройства).

5. Объединить свободные неразмеченные диски ВМ в RAID-массив 5-го уровня.

6. С помощью менеджера LVM: создать физический том на созданном RAID-массиве; создать группу томов (название — по фамилии студента латиницей) и логический том `lv0`. Смонтировать логический том на постоянной основе в `/raid/0`.

7. Средствами менеджера виртуальных машин добавить к ВМ новый диск; добавить диск в RAID-массив и в LVM. Убедиться, что размер логического тома `lv0` увеличился.

8. Пропускаем этот пункт. ~~Подключить диски из папки «Диски» к виртуальной машине. Восстановить целостность RAID-массива, смонтировать в `/raid/1` и найти в представленных файлах строку, начинающуюся с `flag<`, в которой указано следующее задание.~~

9. Настроить доступ по NFS к каталогу `backup` на подключённом RAID-массиве (например, `/raid/0/backup`).

10. Настроить резервное копирование содержимого домашнего каталога пользователя (`/home/<user>`) в каталог `backup` на сервере раз в 2 минуты, используя cron.

**Часть 2 (на той же ВМ; вторую машину не разворачивать)**

11. Создать правила iptables: заблокировать входящий трафик на порт 22 (SSH), трафик с выбранного IP (например, хост), ICMP (ping). Затем удалить эти правила (задание 12).

12. Удалить правила iptables, добавленные в задании 11.

13. Включить брандмауэр ufw и создать те же правила блокировки.

14. Мониторинг и диагностика: использовать `ss` и `nc` для просмотра портов и проверки доступности.

15. Использовать tcpdump для анализа трафика: фильтр по порту, по ICMP, по IP; создать файл с записью трафика.

16. Использовать iptraf-ng для мониторинга трафика в реальном времени.

17. На сервере настроить дополнительный сетевой интерфейс через netplan, статический IP 172.23.0.7/24.

18. В ufw открыть порты 139 и 445. Проверить доступность портов извне виртуальной машины с помощью `nc`.

19. Установить OpenVPN, настроить сервер: VPN-сеть 10.8.0.1/24, создать конфигурационный файл для одного клиента (проверка — с той же ВМ или с хоста при наличии).

20. Установить и настроить Samba-сервер: сделать доступной по SMB любую директорию, создать тестовый файл в общей папке. Проверить доступ по 172.23.0.7 с той же ВМ или с хоста.

21. Настроить проброс портов 139 и 445 из сети 172.23.0.0/24 в VPN-сеть 10.8.0.0/24.

## Шаблон отчёта

1. **Среда выполнения**
   - Гипервизор (VirtualBox / VMware), версия.
   - Образ: Ubuntu Server (версия).
   - Количество и размер дополнительных дисков.

2. **Задание 1. Развёртывание ВМ**
   - Скриншот или описание: ВМ с подключёнными дисками.

3. **Задание 2. SSH по ключу**
   - Команды настройки; скриншот успешного входа по SSH.

4. **Задание 3. Сеть (netplan)**
   - Конфигурация netplan; выводы `ip a`, `ip r` после `netplan apply`.

5. **Задание 4. Конфигурация дисков**
   - Выводы `lsblk`, `fdisk -l` (или аналог); перечень дисков для RAID.

6. **Задание 5. RAID 5**
   - Команды создания массива; вывод `cat /proc/mdstat` и `mdadm --detail /dev/md0`.

7. **Задание 6. LVM и монтирование `/raid/0`**
   - Команды создания PV, VG, LV; вывод `pvs`, `vgs`, `lvs`; фрагмент `/etc/fstab`.

8. **Задание 7. Расширение RAID и LVM**
   - Команды добавления диска в RAID и LVM; вывод `lvs` и `df -h` до/после.

9. **Задание 8. Восстановление RAID и поиск флага** *(если выполнялось)*
   - Команды сборки массива и монтирования; найденная строка `flag<...>`.

10. **Задание 9. NFS**
    - Содержимое `/etc/exports`; команда проверки монтирования с клиента.

11. **Задание 10. Cron и бэкап**
    - Скрипт резервного копирования (или путь к нему); строка из `crontab -l`; скриншот или вывод каталога `backup` с созданными копиями.

12. **Часть 2. Задания 11–21**
    - iptables/ufw: правила и их удаление (11–13).
    - ss, nc, tcpdump, iptraf-ng (14–16).
    - netplan 172.23.0.7/24 (17), ufw 139/445 и проверка nc (18).
    - OpenVPN: сервер и клиентский конфиг (19).
    - Samba: конфиг, общая папка, проверка доступа по 172.23.0.7 (20).
    - Проброс портов 139, 445 в VPN (21).

13. **Выводы**
    - Кратко: что сделано, с какими трудностями столкнулись.
