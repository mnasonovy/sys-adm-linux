# Лабораторная работа № 3. KVM и Docker

**Среда:** Linux-хост с аппаратной виртуализацией и KVM/libvirt (при необходимости — ВМ с виртуализацией Intel VT-x/AMD-V). **Гостевая ВМ:** Ubuntu Server. Можно использовать ВМ и пользователя из ЛР № 1–2 как основу для хоста KVM.

**Требование:** по каждому этапу (KVM, гостевая ВМ, SSH, диск и файловая система, Docker, контейнер nginx) в отчёт должны входить скриншоты или выводы ключевых команд.

---

## Задания

1. **Подготовить хостовую машину с KVM**
   - Развернуть (или использовать готовый) Linux-хост с поддержкой аппаратной виртуализации.
   - Установить `qemu-kvm`, `libvirt-daemon-system`, `libvirt-clients`, при возможности `virt-manager`.
   - Проверить работу `libvirtd` и команду `virsh list --all`.

2. **Создать гостевую ВМ в KVM**
   - Создать ВМ с Ubuntu Server (через `virt-install` или `virt-manager`).
   - Выполнить базовую установку ОС.

3. **Создать пользователя `user` в гостевой ВМ**
   - Добавить пользователя `user`.
   - Проверить, что пользователь присутствует в системе и может входить.

4. **Настроить SSH по ключу для пользователя `user`**
   - Установить и включить OpenSSH-сервер.
   - Настроить вход по ключу для `user`.
   - Отключить парольную аутентификацию; убедиться, что вход по ключу работает.

5. **Изучить конфигурацию гостевой ВМ**
   - Зафиксировать выводы `lscpu`, `free -h`, `lsblk`, `df -h`, `ip a`, `ip r`.
   - Кратко описать CPU, RAM, диски и сеть гостевой ВМ.

6. **Пробросить дополнительный диск в гостевую ВМ через KVM**
   - На хосте создать дополнительный виртуальный диск (например, 10 ГБ, формат qcow2).
   - Подключить диск к гостевой ВМ (через `virsh attach-disk` или интерфейс `virt-manager`).
   - Убедиться, что в гостевой ВМ появился новый диск (например, `/dev/vdb`).

7. **Создать GPT-разметку, раздел и файловую систему ext4 на новом диске**
   - Создать таблицу разделов GPT и один раздел.
   - Создать файловую систему ext4 на новом разделе.
   - Смонтировать раздел в `/disk` и настроить постоянное монтирование через `/etc/fstab`.

8. **Обеспечить доступность `/disk` пользователю `user`**
   - Настроить владельца и права на `/disk`, чтобы `user` мог создавать и изменять файлы.
   - Проверить создание тестового файла под пользователем `user`.

9. **Установить Docker в гостевой ВМ**
   - Установить Docker (`docker.io` или Docker Engine).
   - Включить и проверить сервис Docker.
   - Добавить пользователя `user` в группу `docker` и убедиться, что он может запускать контейнеры (например, `hello-world`).

10. **Развернуть контейнер с nginx, который выводит фамилию студента**
    - Создать на диске, смонтированном в `/disk`, каталог с HTML‑страницей (`index.html`), в которой явно указана фамилия студента.
    - Запустить контейнер nginx, смонтировав этот каталог внутрь контейнера как корень веб‑сайта.
    - Проверить доступность страницы по IP гостевой ВМ (порт 80) с хоста (через браузер или `curl`) и убедиться, что на странице корректно отображается фамилия.

---

## Шаблон отчёта

1. **Среда выполнения**
   - Хост KVM: дистрибутив и версия, версии `qemu-kvm`/libvirt (при необходимости).
   - Гостевая ВМ: Ubuntu Server (версия), объём дисков, сетевые настройки.
   - IP гостевой ВМ, используемый пользователь.

2. **Задание 1. Подготовка хоста KVM**
   - Выводы `virsh version`, `virsh list --all`; статус `libvirtd` (например, `systemctl status libvirtd`).

3. **Задание 2. Гостевая ВМ**
   - Команда `virt-install` или скриншот `virt-manager`; подтверждение загрузки Ubuntu Server.

4. **Задание 3–4. Пользователь `user` и SSH**
   - Команды создания пользователя; фрагмент `sshd_config` (ключи, `PasswordAuthentication`);
   - скриншот или вывод успешного входа по SSH с хоста.

5. **Задание 5. Конфигурация гостевой ВМ**
   - Выводы `lscpu`, `free -h`, `lsblk`, `df -h`, `ip a`, `ip r`.
   - Краткое текстовое описание ресурсов ВМ.

6. **Задание 6. Дополнительный диск**
   - Команды создания qcow2 и подключения (`virsh attach-disk` или эквивалент); вывод `lsblk` в госте.

7. **Задание 7. Разметка и монтирование `/disk`**
   - Команды разметки GPT и `mkfs.ext4`; фрагмент `/etc/fstab`; вывод `findmnt /disk` или `mount | grep disk`.

8. **Задание 8. Права на `/disk`**
   - Команды `chown`/`chmod` (или ACL); тест создания файла от `user`.

9. **Задание 9. Docker**
   - Вывод `docker --version`, `systemctl status docker`; `docker run hello-world` под пользователем `user`.

10. **Задание 10. Nginx в контейнере**
    - Путь к каталогу на `/disk`, фрагмент `index.html` с фамилией;
    - команда `docker run` (или `docker-compose`) с volume;
    - вывод `curl http://<IP_гостя>/` с хоста или скриншот страницы в браузере.

11. **Выводы**
    - Кратко: что сделано, с какими трудностями столкнулись (вложенная виртуализация, сеть, права Docker и т.д.).

## Критерии оценивания

| Задание | Критерий |
|--------|----------|
| 1 | Хостовая машина подготовлена, KVM и libvirt установлены, `virsh list --all` выполняется без ошибок |
| 2 | В KVM создана и успешно загружается гостевая ВМ с Ubuntu Server |
| 3 | В гостевой ВМ создан пользователь `user` и может войти в систему |
| 4 | Для `user` настроен SSH по ключу, парольная аутентификация отключена |
| 5 | Описаны основные параметры гостевой ВМ (CPU, RAM, диски, сеть) с примерами выводов команд |
| 6 | В гостевую ВМ проброшен дополнительный диск, он отображается в `lsblk` как отдельное устройство |
| 7 | На новом диске создан раздел с GPT, файловая система ext4 и постоянное монтирование в `/disk` через `/etc/fstab` |
| 8 | Пользователь `user` имеет права на работу в `/disk` (создание, чтение и удаление файлов) |
| 9 | В гостевой ВМ установлен и запущен Docker; пользователь `user` может запускать контейнеры (например, `hello-world`) без ошибок прав доступа |
| 10 | Запущен контейнер с nginx; страница доступна по сети, **содержит фамилию студента**, а статические файлы сайта размещены на диске, смонтированном в `/disk` |

Лабораторная работа засчитывается при выполнении всех заданий и наличии отчёта с командами, конфигурационными файлами и выводами.
